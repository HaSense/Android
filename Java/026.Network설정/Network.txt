
WSL과 Windows 그리고 Android 네트워크 설정

1. Windows11에서 wsl를 설정하고 네트워크를 구성하면 
   Windows Console ---> wsl      ping으로 쉽게 연결된다.
   네트워크 어답터는 Hyper-V Virtual Ethenet Adpter를 사용한다.
   wsl ip는 ifconfig 명령으로 확인해 보면 172.25.240.204 경우가 많다.
   Windows의 경우는 내부망에 잡혀있을 경우 192.168.0.x가 되며 
   공인 ip로 잡혀있다면 작업하기 수월하게된다.

2. wsl --> Windows가 ping이 안되는 경우가 많다. 
   wsl은 10.x.x.x로 시작하는 NAT망의 가상망일 확률이 높기 때문이다.
   이때 windows 쪽으로 접근하기 위해서는 /etc/resolv.conf에 있는 nameserver ip를 확인하라.
   windows의 가상망 ip를 확인할 수 있고 wsl은 nameserver의 ip로 접근할 수 있다.
   이때 접근은 ping으로는 네트웍 레벨이 차이 나서 접근할 수 없고 curl이나 nc 명령어로 접근할 수 있다.

   windows console에서 
   >python -m http.server 9999
   명령어로 웹서버를 구동시켜 보아라.
   curl 또는 nc 명령어로 windows ip:9999포트에 접근해보자.
   
   $> curl http://$(ip route show | grep -i default | awk '{ print $3 }'):9999
   또는 windows nameserver ip가 10.255.255.254일때
   $> curl http://10.255.255.254:9999 실패시 기본 게이트웨이 ip를 사용하라.
   $> curl http://172.25.240.1:9999

   $> nc -zv $(ip route | grep default | awk '{print $3}') 9999
   $> nc -zv http://172.25.240.1:9999

3. 포트포워딩
   윈도우에서 가능 powershell에서 명령어

   -현재 포트포워딩 목록 확인
   > netsh interface portproxy show all
   
   -포트 포워딩 설정 명령어
   > netsh interface portproxy add v4tov4 listenport=8000 listenaddress=0.0.0.0 connectport=8000 connectaddress=172.25.240.204

   -포트 포워딩 해제 명령어
   > netsh interface portproxy delete v4tov4 listenport=8000 listenaddress=0.0.0.0
